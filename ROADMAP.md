# ReplayCam 功能規劃

> 記錄目前想做、可能做、未來考慮的方向，依優先度與難度分類。

---

## 🟢 短期（低難度、高價值）

### UI / UX
- [ ] **旋轉後自動重置小視窗位置**：切換橫/直向時，小視窗可能飄到畫面外，應自動歸位到安全區域
- [ ] **延遲秒數大字提示**：切換延遲時，畫面中央短暫顯示「5 秒」等大字，確認已切換
- [ ] **儲存成功顯示縮圖**：儲存完成後短暫顯示影片第一幀作為預覽，而非只有文字提示
- [ ] **深色/淺色模式適配**：控制列背景色跟著系統主題調整

### 相機功能
- [ ] **前鏡頭支援**：切換前/後鏡頭（適合自拍練習、舞蹈）
- [ ] **手電筒開關**：低光源環境下輔助補光
- [ ] **縮放控制**：兩指捏合縮放（目前固定焦距）

---

## 🟡 中期（需要一定開發量）

### 訓練輔助
- [ ] **慢動作回放**：儲存時可選 0.5× / 0.25× 速度輸出，看清楚動作細節
- [ ] **比較模式（左右並排）**：畫面分割，左邊顯示當前延遲畫面，右邊顯示上一次儲存的片段
- [ ] **網格疊層**：在延遲畫面上顯示格線（九宮格 / 對稱軸），幫助觀察身體對稱與站位
- [ ] **計時器 / 倒數**：設定幾秒後自動截取當下畫面或開始緩衝，方便自拍練習

### 儲存與管理
- [ ] **App 內片段庫**：儲存的片段直接在 App 內瀏覽，不用跳去相簿
- [ ] **快速標記**：按一個按鈕標記「這一刻」，只儲存前後 3 秒的短片段（不用每次手選長度）
- [ ] **語音備註**：儲存影片時，同步錄製一段教練或自己的語音說明，合在同一個影片檔

### 效能
- [ ] **背景緩衝保持**：App 進入背景時維持緩衝（目前進背景相機會停）
- [ ] **儲存進度條**：匯出影片時顯示實際百分比，而非只顯示「儲存中...」

---

## 🔴 長期 / 進階（高難度、需研究）

### 動作分析
- [ ] **姿勢關節偵測**：用 Vision framework 標出關節點（肩、肘、膝、踝），計算角度
- [ ] **動作角度量測**：點選兩個關節，即時顯示夾角度數（例如：膝蓋彎曲 120°）
- [ ] **動作週期計數**：偵測重複動作（投籃、揮桿、跳繩），自動累計次數
- [ ] **畫面標注工具**：可在延遲畫面上用手指畫線、標箭頭，截圖後分享給教練

### 多裝置
- [ ] **雙機位同步**：兩支 iPhone / iPad 透過 Wi-Fi 或 Bluetooth 同步延遲，一台拍正面、一台拍側面
- [ ] **AirPlay 輸出**：將延遲畫面投放到大螢幕（教室、訓練場），讓多人同時觀看

### 個人化
- [ ] **運動情境預設**：選擇「高爾夫」「棒球投球」「游泳換氣」等，自動設定最佳延遲秒數與建議拍攝角度
- [ ] **個人訓練紀錄**：加入日期、標籤（動作名稱、訓練目標），方便追蹤進步

---

## 🔧 技術債 / 已知問題

| 項目 | 說明 |
|------|------|
| 旋轉後小視窗可能超出畫面邊界 | 需加入邊界夾緊（clamp）邏輯 |
| App 進背景後相機停止 | 需處理 `UIApplication.didEnterBackgroundNotification` |
| 長時間使用記憶體會持續增長 | 緩衝區有上限但 UIImage 物件釋放時機需確認 |
| 匯出進度沒有細粒度回報 | `requestMediaDataWhenReady` 不直接提供進度，需額外計算 |
| Git commit author 未設定 | `git config --global user.name / user.email` 需補上 |

## 💡 靈感備忘

- **延遲時間可視化**：用一條時間軸顯示「現在在哪、延遲到哪」，類似影片播放列
- **教練模式 vs 選手模式**：教練模式可標注、暫停、倒轉；選手模式介面極簡只有延遲畫面
- **AR 疊層**：用 ARKit 在人體上疊加骨架或軌跡線（非常長期）
