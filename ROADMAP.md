# ReplayCam 功能規劃

> 記錄目前想做、可能做、未來考慮的方向，依優先度與難度分類。

## ✅ 已完成

### 架構 / 效能
- [x] **拆分檔案**：將原本 1293 行的單一 ContentView.swift 拆成 6 個獨立模組
- [x] **Thread safety**：FrameBuffer 用 NSLock 保護，修正 data race
- [x] **CIContext 重用**：避免每幀重建，改為初始化時建立一次
- [x] **非同步匯出**：VideoExporter 改用 `requestMediaDataWhenReady` + async/await
- [x] **幀率上限**：鎖定 30fps，避免新機型自動選 60fps 導致 CPU 加倍
- [x] **記憶體控管**：`autoreleasepool` + 每 300 幀清一次 `ciContext.clearCaches()`
- [x] **延遲解碼獨立佇列**：新增 `decodeQueue` 避免 frameQueue 被阻塞

### 相機功能
- [x] **縮放控制**：即時視窗支援雙指捏合放大／縮小（0.5× ~ 3×）
- [x] **旋轉方向修正**：橫向錄影不再旋轉 90°，自動對應裝置方向

### UI / UX
- [x] **小視窗可拖移**：即時畫面可自由拖到畫面任意位置
- [x] **旋轉後小視窗歸位**：切換橫直向時自動 clamp 回安全區域
- [x] **延遲滑軌**：1–30 秒連續可調，取代原本固定選項
- [x] **儲存長度滑軌**：3–30 秒可調，直接取代確認對話框
- [x] **自動收合控制列**：點擊畫面切換顯示 / 隱藏，收合時顯示當前延遲秒數
- [x] **即時畫面 LIVE 角標**：取代原本大紅底「即時畫面」文字
- [x] **狀態內嵌於儲存按鈕**：就緒 / 準備中 / 儲存中 整合在一個按鈕內
- [x] **毛玻璃控制列**：改用 `.ultraThinMaterial`，視覺更輕盈

## 🟡 中期（需要一定開發量）

- [ ] **比較模式（左右並排）**：畫面分割，左邊顯示延遲畫面，右邊顯示上次儲存片段

### 儲存與管理

## 💡 靈感備忘

- **延遲時間可視化**：用一條時間軸顯示「現在在哪、延遲到哪」，類似播放進度列
- **教練模式 vs 選手模式**：教練模式可標注、暫停、倒轉；選手模式介面極簡只有延遲畫面
- **AR 疊層**：用 ARKit 在人體上疊加骨架或軌跡線（非常長期）
- **Widget / Dynamic Island**：快速啟動相機或顯示目前延遲秒數
